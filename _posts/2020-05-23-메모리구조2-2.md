# 메모리 구조

프로그램이 실행되기 위해서 프로그램이 메모리에 로드되어야 합니다.

또한 프로그램에서는 변수들을 저장할 메모리도 필요합니다.

따라서 운영체제는 프로그램의 실행을 위한 다양한 메모리 공간을 제공하고 있는데

대표적인 메모리 공간으로는 아래와 같습니다

1. **코드(code)영역**
2. **데이터(data)영역**
3. **힙(heap)영역**
4. **스택(stack)영역**

![](https://i.imgur.com/FdJ8Xbd.png)

<hr/>

+ ###  데이터 영역

  + 코드(Text)영역은 ***실행할 프로그램에 코드*** (프로그래머가 작성한 소스코드) 가 저장되는 영역입니다. 프로그램이 시작하고 끝날때까지 메모리에 계속 남아 있습니다.  실제 연산하는 ***CPU는 코드영역에 저장되어 있는 명령어를 가져다 처리를 합니다.*** 즉, 코드 영역은 컴퓨터가 이해할 수 있는 ***기계어로 번역된 소스코드가 올라가는 공간*** 이며 상수도 여기에 들어갑니다.  



---

+ ### 데이터 영역

  + 메모리의 데이터영역은 프로그램의 ***전역 변수와 정적(static) 변수가 저장되는 영역***입니다.

    프로그램이 시작하며 메모리에 적재되고 종료가되면 사라지는데 이는 곧 전역변수와 정적변수는 프로그램이 종료될때까지

    계속 남아서 다른 함수에서도 사용할 수 있는 이유가 됩니다.

---

+ ### 스택 영역

  + 메모리의 스택 영역은 함수의 호출과 관계되는 ***지역변수와 매개변수***가 저장되는 공간입니다.

    스택 영역은 함수가 호출할때 할당되며 호출이 완료되면 데이터는 사라지게 됩니다.

    스택영역은 박스가 쌓이듯이 먼저 들어오는 순서대로 저장이되며 호출시에는 마지막에 들어오는 데이터가 먼저 호출 됩니다(Last In First Out), 이렇게 메모리도 높은 주소에서 낮은 주소의 방향으로 할당 되게 됩니다.

    

    스택영역에 저장되 있는 함수의 호출 정보를 Stack Frame이라고 부르며, 프로그램에서 현재 실행중인 함수에 대한 정보를

    저장하는 스택 자료구조를 Call Stack 이라고 부릅니다.

    Stack 영역이 클수록 Heap영역이 작아지고 반대로 Heap 영역이 클수록 Stack 영역이 작아지게 되는데 스택 영역은 컴파일시 크기가 결정되게 됩니다.

---

+ ### 힙 영역

  + 힙 영역은 사용자에 의해 프로그램이 진행시 메모리를 사용할 수 있는 영역입니다. 사용자가 원하는 시점에 할당하고 소멸할수있또록 변수들이 할당되는 영역입니다.

    스택에서 포인터 변수를 할당하고 포인터가 가리키는 Heap 영역의 임의공간으로부터 동적 할당한 크기만큼 사용합니다.

    메모리를 사용하고 난 후에는 반드시 메모리 반환을 해줘야하는데 C,C++ 같은 경우에는 개발자가 직접해야 하지만 JAVA같은 경우에는 Garbage Collection이 개발자 대신 해줍니다.

---

**스택영역과 힙영역에 빈공간**

이미지에 빈 공간은 **Stack** 과 **Heap** 은 여유공간을 위해 지원합니다.

Stack의 지역변수는 사용되고 소멸되기 때문에 용량의 불확실성 때문에 밑에서부터 올라가게 되는데, 이때 Stack영역이 Heap영역으로 침범하는 경우를 **Stack overflow**라 부릅니다.

Heap도 같은이유로 위에서부터 채워가게 되는데 마찬가지로 Heap도 Stack영역으로 침범하는 경우를 **Heap overflow**라고 부릅니다.
