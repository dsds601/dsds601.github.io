# 메모리

***메모리*** 는 **주소와 데이터**로 구성이 되어있다



1. 주소공간

   + 프로세스에 참조할 수 있는 주소들의 범위를 의미한다. 

     프로세스와 1:1 관계를 가지며 사용자 쓰레드는 주소공간을 공유한다.

     주소공간의 크기는 CPU의 주소버스에 의존을 하며 CPU는 컴퓨터에 메모리에 설치되며 각 메모리의 주소 공간이 있다. 이렇게 설치된 메모리의 주소공간을 **물리 주소 공간**이라고 한다. 물리 주소 공간은 하드웨어 입장에서 바라본 공간으로 컴퓨터마다 그 크기가 다르다. 이와 반대로 사용자 입장에서 바라본 주소 공간은 **논리 주소 공간**이라고 한다.

2. 물리 주소

   + 컴퓨터의 메인 메모리를 접근할 때 사용되는 주소를 물리 주소 라고 한다. 이는 메모리 관리자 입장에서 바라본 주소이다. 메모리에서 어떤 값을 읽어내기에는 편하지만 운영체제가 업그레이드 되면 주소 범위가 바뀔수 있기에 사용자 입장에서는 불편할 수 있다.

3. 가상 주소

   + 물리주소는 사용자입장에서 범위가 바뀌면 불편할 수 있기에 프로세스 입장에서 사용자 주소를 따로 정의하는데 이를 가상주소 라고 한다. 물리주소와 관계없이 0번지에서 시작한다.



CPU는 레지스터를 참조하며 메모리 공간을 보호하고 레지스터 정보는 PCB에 담겨있다.

개별적인 메모리 공간을 분리하고 프로세스가 합법적인 영역만 접근 하도록 하는것이 필요하다. 이때 레지스터는 base와 limit으로 나눠 보호 기법을 제공한다

base는 프로세스가 합법적인 물리 메모리 주소의 값을 저장하고 limit는 주어진 영역의 크기를 저장한다. 즉 프로세스가 사용할 수 있는 가장 큰 주소는 base와 limit의 합이된다.

## 주소 할당 (Address Binding)

CPU가 명령을 수행하면  data를 메모리 어느 번지에 들어갈지 결정하는데 이걸 **주소할당**이라고 부릅니다. 주소를 할당하는데에는 크게 3단계가 있습니다.

1. **compile time**

   + 캄파일 할 때 바인딩이 이루어집니다. 물리적인 메모리가 많이 비었어도 주소가 결정 되었으면 변경할 수 없는 비효율적인 방법으로 주소를 변경할수 없으므로 컴파일된 코드를 absolute code라고 부릅니다.

2. **load time**

   + 프로그램이 실행되었을때 할당이 이루어지며 컴파일러가 재배치 가능한 코드라 하여서 relocatable code라고 부릅니다.

3. **execution time**

   + 현재 사용하는 방법으로 프로그램이 실행된 후에도 주소를 변경 가능합니다.

     CPU가 주소를 참조할 때마다 새로 할당 상태를 점검해야  하므로 MMU같은 하드웨어 적인 지원이 필요합니다

### MMU (Memory Management Unit)

---



![MMU](https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2FbJDuGg%2FbtqxC7uysgv%2Fu6kxrknlZw494q7w8uz6K0%2Fimg.png)

- 이미지 출처:https://dely.tistory.com/78

CPU에서 논리주소를 사용하는 경우 CPU와 Memory 사이에서 Translation이 일어나게 되는데 이 Translation을 수행하는 장치를 MMU(Memory Management Unit)이라고 합니다.

CPU로부터 받은 가상주소를 물리주소로 번역하여 메모리에 보내는 역활을 합니다.
