* ####  SQL(Structured Query Language) 이란?

  * **RDBMS**에 접근해서 데이터의 [입력],[수정],[삭제],[검색] 하는 등의 기능을 가진 **RDBMS** 관리 언어입니다.

  * **<참고>** DB를 배운다는건 SQL 언어를 배우는 것과 동일합니다.

  * 특징 [입]

    * [절차적 언어]가 아닌 [구조적 언어]이다.

      **<참고>**

    * 절차적(procedural) 언어 : 모든 처리 과정을 일일이 기술하고 기술된 순서대로 로직이 처리되는 언어. (C, Java 등)

    * 구조적(structured) 언어 : 처리 과정을 일일이 기술할 필요 없이 일정한 틀이나 패턴이 있어 맞게 조건들만  나열하면 로직이 처리되는 언어. (SQL등)

  

* SQL 종류 (기 , 입)

  * DDL(Data Definition Language = 데이터 정의 언어)

    * =>객체 생성 수정 삭제 등의 작업을 한다.

    * CREATE -> 객체생성

    * ALTER -> 객체수정                **※UPDATE 가 아니다!**

    * DROP -> 객체삭제 ...등등         **※DDL은 트랜잭션이 안걸린다.**

      **<주의>** 오라클에서 객체란?

      이름을 가지고 저장되는것을 객체라 부른다.                **!자바와 다름!**

      **<참고>** 오라클에서 객체는 [테이블] , [뷰] , [트리거] , [인덱스] ,[ 프로시저] , [시퀀스] 등 을 말한다	 **※데이터는 객체가 아니다.**

      > ※테이블 제거시 유의 사항※

      * 참조 하고 참조 당하는 관계에서 참조하는 테이블 먼저 지우고 당하는 테이블을 지워야합니다
        PK 먼저 지우고 FK 지웁니다.

      

  * DML(Data Manipulation Language = 데이터 처리 언어)

    * **객체 [테이블]** 안의 데이터를 입력,수정,삭제,검색 등의 작업을 한다.

      * INSERT -> **행** 단위의 데이터 입력          **※셀 단위 데이터 입력은 안된다!**

    * UPDATE ->  **셀** 단위의 데이터 수정/삭제     **※ALTER 가 아니다.**

    * DELETE ->  **행**단위의 데이터 삭제

    * SELECT -> n행 m열의 데이터 검색

    * ->DB실력은 SELECT 구사 능력이다.

      * **<문>**셀에 들어있는 데이터를 제거하는 SQL 구문은?

        	-> UPDATE!

      * **<문>**행에 들어있는 데이터를 제거하는 SQL 구문은?

        	-> DELETE!

        

  * DCL(Data Control Language = 데이터 제어 언어)

    * COMMIT -> 트랜잭션 작업 단위의 데이터 입력 , 수정 , 삭제 작업을 모두 **인정**
    * ROLLBACK -> 트랜잭션 작업 단위의 데이터 입력 , 수정 , 삭제 작업을 모두 **취소** 복구함.
    * GRANT -> 접근제어 , 작업권한 **허용**
    * REVOKE -> 권한 **제거**

* #### 트랜잭션(Transaction) 이란? [입 실 기]

  * **정의 :** 모두 취소되거나 모두 완료되는 2가지 결과로만 진행되는 작업 단위를 말합니다.
  * 데이터에 무결성을 위해 작업 합니다.
  * 트랜잭션은 주로 **입력 **또는 **수정 **또는 **삭제** SQL 구문 실행 시 설정합니다.
  * 트랜젹션이 걸린 작업은 **COMMIT (실행)** 또는 **ROLLBACK (취소)** 을 해야 실제 작업이 되고, COMMIT이나 ROLLBACK을 안하면 모두 가상 작업 입니다.
  * **SELECT** 같은경우에는 **검색**만을 이용하기에 DB에 영향을 안미치기에 트랜잭션을 걸 필요가 없습니다.
  * DDL은 트랜잭션을 걸 수 없습니다. 자체적으로 CREATE , ALTER 는 COMMIT , ROLLBACK 들어갑니다.
  * **<주의>**  트랜잭션은 의도를 가지고 설정하는 것!
  * **<주의>**  트랜젹션이 없는 작업은 N가지 이상의 다양한 결과로 진행 될 수 있다.

* #### 오라클에서 [계정] 이란?

  * 오라클에 접속하는 사용자를 말한다.
  * 사용자마다 권한을 다르게 줄 수 있습니다.
  * 오라클은 다수의 클라이언트에게 동시에 정보를 제공하는 서버이므로 서버의 특징인 계정과 암호를 가지고 있다.

* 오라클 계정의 종류

  * SYS -> 오라클 설치 시 기본적 으로 만들어지는 계정 모든 것을 할 수 있다.   **※최고 계정※**

  * SYSTEM -> 오라클 설치 시 기본적으로 만들어지는 계정 , 모든 것을 할 수 있다. **단! [데이터베이스] 생성 권한이 없다.**

  * SCOTT -> 오라클 설치 시 기본적으로 만들어지는 계정, 권한이 거의 없다.

  * 생성 계정 -> 계정 생성 권한을 가진 계정이 만드는 계정으로 계정별 권한이 설정된다.

* [계정]이 생성한 모든 **[객체]** 들을 의미 한다. (오라클 객체 =>테이블 , 뷰 ,인덱스 , 프로시저 , 트리거 등)

  * 객체들이 DB구조에 대한 정의와 이에 대한 제약조건등을 기술한 [명세서]이기 때문에 객체도 스키마 이다. 
    * **EX) [Kim]이란 계정으로 (테이블 , 뷰 ,인덱스 , 프로시저 , 트리거 ) 객체를 만들면 모두를 가리켜 [Kim 스키마] 라고 부릅니다.**

  예)abc 계정 -> employee 테이블 생성       abc.employee 형태로 저장

  예)kim 계정 -> employee 테이블 생성       kim.employee 형태로 저장

  예)kim 계정 -> employee 테이블 생성       kim.employee 형태로 저장안됨 , 이미 생성되어 있음

  

#### 데이터베이스 차원에서 테이블(table) 이란?

* RDBMS에서 데이터가 실질적으로 저장되는 [논리적 장소( = S/W적인 장소)]를 말합니다.

* 컬럼(column = 열)과 로우(row = 행) 로 구성 되어 있습니다.

* 테이블은 이름을 가지고 저장되는 객체에 한 종류 입니다.

* 개발자 입장에서는 손이 제일 많이 가는 객체입니다.

  ####                                                         테이블 명령어

  |    table    |   명령어   |                      |
  | :---------: | :--------: | :------------------: |
  | 데이터 입력 |   insert   |         into         |
  | 데이터 수정 |   update   |   테이블 , 객체명    |
  | 데이터 삭제 |   delete   |         from         |
  | 데이터 검색 | **select** |                      |
  | 테이블 삭제 |    drop    | 객체명 (table ,view) |



#### 테이블(table)을 생성하는 SQL 구문의 형식

---------------

=================================

CREATE TABLE 테이블명(                              **컴마를 앞에 두는게 에러를 찾는데 편하다.**

```
			컬럼명1     자료형      제약조건

		   ,컬럼명2     자료형      제약조건

		   ,   ~        ~          ~
	    	,컬럼명n     자료형      제약조건
```

);

===================================

CREATE TABLE 테이블명(

```
			컬럼명1     자료형      제약조건,

		    컬럼명2     자료형      제약조건,
              ~        ~            ~    ,

			 컬럼명n     자료형      제약조건
```

);

===================================

* 자료형(Data type) : 테이블의 컬럼에 입력될 데이터의 유형을 말한다. 문자형 , 숫자형 , 날짜형 등이 있다.

* 제약조건 : 입력되는 데이터의 입력 가능 조건을 말한다. 제약조건에 위배되는 데이터는 삽입될 수 없다 

  **한 행** 자체가 못들어간다. 제약조건이 생략되면 데이터가 입력되지 않을 경우 null이 저장된다.



* **<주의> 테이블명 , 컬럼명 제작 규칙**
  * 영문자, 1~9 , $ , # , _ 로 구성되고 **반드시 영문자로 시작**해야 한다.
  * 30자를 초과할 수 없다. SQL 예약어는 사용할 수 없다.
  * 하나의 계정이 만든 테이블명은 유일해야 하고 하나의 테이블 안에서 컬럼명은 유일해야 한다.
  * 테이블명 , 컬럼명은 가독성 있게 주는 것이 제일 중요하다.
  * 테이블명, 컬럼명 , 제약조건명을 "" 로 감싸거나 ""없이 그냥 써도 된다.
